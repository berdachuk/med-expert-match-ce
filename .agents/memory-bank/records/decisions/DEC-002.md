# DEC-002: Interface + Impl Service Pattern

- **Status:** Accepted
- **Date:** 2026-06-13
- **Module:** all
- **Affects:** all modules
- **Legacy alias:** D-002

## Rationale

Testability, clear modulith boundaries, Spring DI flexibility. Every service has a public interface in `service/` and an `@Service` impl in `service/impl/` with `@Slf4j` + `@RequiredArgsConstructor`.
