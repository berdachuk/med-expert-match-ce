## Summary

Partitioned the memory-bank to prevent merge conflicts when multiple AI agents work in parallel worktrees. Replaced single-writer monolithic Markdown files with append-only JSONL registries + per-record files + generated index files + module locks.

## What was built

- `registry/` — append-only JSONL for REQ/NFR/SCN/TEST/DEC/RISK/TASK IDs + `SCHEMA.md`
- `records/` — one file per milestone/decision (`progress/M{NN}.md`, `active/M{NN}.md`, `decisions/DEC-###.md`)
- `locks/` — per-module ownership claims with `README.md` (coupled-file-pair table)
- `worktrees/` — git-ignored per-branch scratchpads
- `scripts/sync-memory-index.sh` — deterministic regenerator of 5 generated index files + `--check` CI gate
- Migrated DEC-001..014 + recent milestones (M111/122/125/129/130/131/132) as seed data
- Updated `AGENTS.md` Memory Bank section + `bootstrap-new-project.md` template

## Bugs fixed in sync script

- `--check` mode resolved on-disk paths via `$ROOT/.agents/$rel` (wrong base for memory-bank files) → switched to absolute paths
- `jq` `contains($r.id)` substring match → exact `==` match (prevents REQ-001 matching REQ-0010)
- `realpath --relative-to` (not portable) → hardcoded relative paths
- Fragile `productContext.md` splicing → grep-guarded with fresh-file fallback
- `ls | sort` glob parsing → `sorted_glob_desc` helper with `mapfile`-style read loop
- Unused `jsonl_each` dead code removed
- Missing `trap cleanup EXIT` → added (prevents stale `tmp/` on failure)
- `tmp/` not git-ignored → added to `.gitignore`
