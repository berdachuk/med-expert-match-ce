## Summary

Applied two improvements from Spring AI 2.0's composable tool calling architecture:

1. **`AugmentedToolCallbackProvider<AgentThinking>`** — Wrapped all 10 tool objects in both `MedicalAgentConfiguration` (main + subagent ChatClients) and `AgentToolCallingConfiguration` (ToolCallbackResolver chain) with inner-thinking augmentation. The model must now articulate its reasoning (`innerThought`) before each tool call, improving traceability for medical compliance.

2. **Advisor ordering** — Set `SessionMemoryAdvisor` order to `ToolCallingAdvisor.DEFAULT_ORDER + 1` so it sits inside the tool loop, capturing the full tool request/response transcript in session memory. `ToolCallingAdvisor` already had `conversationHistoryEnabled(false)` to avoid duplicate writes.

3. **`ToolSearchToolCallingAdvisor`** — Deferred: this class does not exist in Spring AI 2.0.0 GA. It's a planned/community feature not yet released.
